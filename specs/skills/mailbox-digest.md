# Mailbox Digest

Digest output should help the user understand mailbox activity quickly.

- Group activity by useful human themes: sender, category, urgency, thread, action taken, and anomalies.
- Mention counts and notable changes when they are available, but keep the narrative brief.
- Highlight messages needing user action, unresolved risks, failed automation, and unusual spikes.
- Do not expose raw internal IDs unless the caller specifically needs them.
- Keep summaries grounded in retrieved mailbox data and clearly mark gaps when the period has incomplete sync coverage.
