# Code of Conduct

MailSubsystem is a project for careful work on private email automation. Keep
discussion respectful, specific, and grounded in the safety and privacy impact
of the change being discussed.

## Expected Behavior

- Be respectful of different experience levels and operating environments.
- Assume good intent, but ask for clarification when a change affects mailbox
  safety, privacy, or cost.
- Use synthetic, anonymized, or heavily redacted examples when discussing email
  content.
- Keep review comments focused on the code, behavior, documentation, and
  release risk.
- Accept maintainer decisions about safety boundaries, supported deployment
  shapes, and release timing.

## Unacceptable Behavior

- Harassment, insults, discriminatory language, or personal attacks.
- Posting private email, credentials, logs with secrets, mailbox exports, or
  exploit details in public issues or pull requests.
- Pressuring maintainers to bypass safety review for mailbox mutation, local
  API exposure, or provider-data handling changes.
- Spam, unrelated promotion, or automated low-signal reports.

## Reporting

For conduct concerns, open a minimal public issue asking for a private contact
path, or use the repository security reporting path if private data is involved.
Do not include sensitive details publicly.

Maintainers may edit, hide, or remove comments; close issues or pull requests;
or block participants when needed to protect contributors or users.
